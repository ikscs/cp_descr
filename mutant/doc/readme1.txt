authentication / authorization

use-cases


AP политика авторизации (выполняет супер пользователь)

   настраивает политику средствами UF
   передает tenant_id, ключи в production


CA регистрация клиента (кто выполняет ?)

   завести клиента в БД (customer_id)
   завести пользователя клиента (роли: admin, viewer) в UF
   + заявка на выполннение обработки данных

CU редактирование клиента (выполняет супер пользователь)

   изменить параметры клиента в БД
   + заявка на выполннение или паузу обработки данных

CR удаление клиента (выполняет супер пользователь)

   пометить клиента как удаленного в БД
   + заявка на паузу обработки данных


UM управление пользователями (выполняет пользователь с ролью admin)

   создание, удаление, назначение ролей в UF
   для роли admin - редактирование номера подразделения (если у самого код ЦО)


UP управление паролями пользователя (выполняет пользователь средствами UF)


PE проверка разрешения на выполннение обработки данных


AE проверка разрешения доступа к приложению

   проверка пароля, получение токена и роли из UF - автоматически

   предоставление функций приложения в соответствии с ролью - фронтенд автоматически

   предоставление доступа к своим данным - бекенд автоматически
